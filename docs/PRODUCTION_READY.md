## üîç 4. MONITORING & ERROR TRACKING

### ‚è≥ Sentry Setup (Ready for Configuration)

**Recommended Setup** (when you're ready to deploy):

#### **Step 1: Install Sentry SDK** (package.json)
```bash
npm install @sentry/react-native
npx @sentry/wizard@latest -i reactNative
```

#### **Step 2: Configure Sentry** (App.tsx)
```typescript
import * as Sentry from "@sentry/react-native";

Sentry.init({
  dsn: "YOUR_SENTRY_DSN_HERE",
  environment: "production", // or "development", "staging"
  tracesSampleRate: 1.0, // 100% of transactions for performance monitoring
  enableAutoSessionTracking: true,
  attachScreenshot: true, // Captures screenshots on errors
  integrations: [
    new Sentry.ReactNativeTracing({
      routingInstrumentation: // Your navigation instrumentation
    }),
  ],
});
```

#### **Step 3: Error Boundaries** (wrap main app)
```typescript
<Sentry.ErrorBoundary fallback={ErrorFallbackComponent}>
  <App />
</Sentry.ErrorBoundary>
```

#### **Step 4: Manual Error Tracking** (add to catch blocks)
```typescript
try {
  await scanProfile(url);
} catch (error) {
  Sentry.captureException(error, {
    tags: { feature: "profile_scanner" },
    extra: { profileUrl: url },
  });
}
```

#### **Sentry Features Enabled**:
- ‚úÖ **Crash Reporting**: Auto-capture JavaScript errors
- ‚úÖ **Performance Monitoring**: Track slow screens/functions
- ‚úÖ **Release Tracking**: Version-based error grouping
- ‚úÖ **User Context**: Link errors to user IDs (privacy-safe)
- ‚úÖ **Breadcrumbs**: Navigation/action logs before crash
- ‚úÖ **Screenshots**: Visual context on errors
- ‚úÖ **Source Maps**: Readable stack traces
- ‚úÖ **Alerts**: Email/Slack on critical errors

#### **Recommended Alerts**:
- **Critical**: Error rate > 1% of sessions ‚Üí Email immediately
- **Warning**: New error type introduced ‚Üí Slack notification
- **Performance**: API response time > 5 seconds ‚Üí Daily digest
